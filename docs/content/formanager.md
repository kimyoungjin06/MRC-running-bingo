# 🛠️ 운영진용 매뉴얼 v1.0 (ForManager)

## 1. 운영 목표
- 4주 동안 **참여 지속** + **안전한 겨울 러닝** + **협동 문화 강화**
- 초보도 완주 가능, 고수도 재미(효율/전략/도움) 있게

---

## 2. 사전 준비 체크리스트(Week 0)
### 2.1 티어 확정
- 초보/중수/고수 기준을 미리 정하고 참가자에게 공지
- (권장) 티어는 게임 중 변경하지 않음

### 2.2 인증 채널/도구 준비
- 채널: 단톡/오픈채팅/패들릿 중 1개로 통일
- (권장) 구글시트 1장:
  - 참가자별 “완료 칸 수”
  - 빙고 라인 수
  - Wild 토큰 보유/사용 로그
  - Seal 적용 상태(대상/타입/남은 러닝 수)

### 2.3 카드 운영 방식 결정
- 카드 뒷면(참가자 공개): `코드 + 타입(A/B/C/D/W) + 난이도(★)`
- 카드 앞면(미션 내용): 선택 마감 후 공개(퍼즐감 유지)
- W카드는 일정 영향이 크므로 **선택 전에 내용 공개 권장**

---

## 3. 세팅 운영 절차
### Step 1) 공지(킥오프 메시지)
- README 배포 + 기간/인증 방식/목요미식회 안내
- “안전 우선” 문구 포함

### Step 2) 카드 선택 수집
- 구글폼으로 25장 선택 받기(타입별 고정 구성):
  - A 10 / B 7 / C 5 / D 2 / W 1
- 운영진 검증:
  - 누락/중복/타입 개수 불일치 확인

### Step 3) 카드 공개 & 빙고판 배치
- CardDeck 공개(또는 개인별 선택 카드만 공개)
- 참가자가 5×5 배치 후 제출
- 운영진 검증:
  - 중앙 W 배치
  - C 상하좌우 인접 금지
  - D 모서리 금지

---

## 4. 진행 운영(Week 1~4)
### 4.1 주간 운영 루틴(권장)
- 월: “이번 주 테마/리마인드”(선택, 분위기용)
- 목: 목요미식회 참여 독려(강요 X)
- 주말: 벙개 추천 / 주간 하이라이트 공유
- 매주 1회: 리더보드(완료 칸 수) 업데이트

### 4.2 인증 판정 최소화(분쟁 방지)
- 기본은 자기신고 OK
- 아래 3개만 운영진 ✅ 확인 권장
  1) 3칸 체크 주장
  2) Wild 달성
  3) Seal/Shield 선언

---

## 5. Wild 토큰(Seal/Shield) 운영 팁
### 5.1 기록 방식
- 시트에 “토큰 보유 여부”와 “사용 로그(날짜/대상/타입/남은횟수)” 남기기

### 5.2 Seal 적용 관리
- Seal 선언 시 즉시:
  - 대상자 태그 확인
  - 봉인 타입(B 또는 C)과 “다음 2회” 명시
- 대상자가 Shield 쓰면:
  - Seal 무효 처리(공격/방어 토큰 모두 소모 권장)

### 5.3 감정 이슈 방지 장치(권장 운영 규정)
- Seal 타겟은 “진행도가 더 높은 사람” 또는 “Top3”만 허용(룰북 준수)
- 동일 대상 중복 타겟 금지
- Seal은 “플레이를 멈추게” 하지 않도록 타입 봉인만(러닝 불인정 금지)

---

## 6. 이벤트/부문상(게임 경험 강화)
### 6.1 중간 목표(빙고 라인)
- 1빙고/2빙고/3빙고 달성자 “주간 공지 + 칭호”
- 부문상 예시
  - 완주 1등(시간)
  - 협동왕(C카드 최다)
  - 페이스메이커상(도움 러닝)
  - 목요미식회 개근상
  - 초보 성장상(초보 중 완주/최다 참여)

### 6.2 운영진이 만들 수 있는 “하이라이트”
- 주말 1회 “크루런 데이”(많이 모이는 날) 기획 → 협동 분위기 강화
- 단, 특정 W조건과 결부시킬 경우 참여 가능자 고려

---

## 7. 흔한 예외 케이스 처리
- **하루 2번 러닝**: 앱 활동 2개면 각각 1회로 처리(각각 최대 3칸)
- **날씨 조건 애매**: 날씨앱 캡처 우선, 안전 우선
- **같이 뛰었는지 애매**: 태그/사진/동시 인증 우선
- **D(누적/연속) 인정 기준**: 마지막 조건을 만족한 러닝 기록에 체크(총 3칸 제한)
